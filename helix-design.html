Part 2

Database Design

A relational schema shaped by five forces: strict multi-tenancy (a workspace is the tenant boundary), append-only server-ordered conversations, structural branching (O(1) forks), policy-as-data RBAC, and polyglot persistence (SQLite dev · PostgreSQL prod). Key Design Decisions

O(1) fork

A branch is a pointer (fork_node_id), not a copy. Reading walks parent_id — Git-style structural sharing.

Append-only nodes

Immutable messages stamped with a monotonic seq → one authoritative order, trivial real-time fan-out.

Policy as data

The permissions table is editable per workspace — an Owner retunes RBAC without a redeploy.

Tenant isolation

Workspace-scoped queries in code and PostgreSQL Row-Level Security in prod — defence in depth.

Polyglot & 3NF

One schema on SQLite (dev) and Postgres (prod). Third normal form, with four deliberate cached denormalisations.

Built vs planned. users · workspaces · memberships · invites are implemented (SQLAlchemy async, created on startup; enums validated in application code today, CHECK constraints land with the Postgres/Alembic migration). The remaining seven tables are designed and arrive with their milestones (M2–M9).

Part 4

Algorithm Design

The load-bearing algorithms, each as a flowchart and/or structured pseudocode with a complexity and correctness note. Two are built today (RBAC, invites); the rest are designed for their milestones.

A1 · O(1) Fork & cross-branch history◻ M4

The signature algorithm. Forking copies no history; the cost moves to a read-time walk up the spine.

Fork — write path

↓

↓ yes

↓

History — read path

resolve_history(branch)
↓
node ← branch.head_node
↓
node ≠ NULL ?
↺ yes
prepend(node) · node ← node.parent (crosses branch boundary)
↓ no
return nodes, root → head

Complexity: write O(1) · read O(n) on the path. Correct because nodes are immutable and parent_id forms a tree — a branch only names a tip + a fork point.

A3 · Recursive Deep-Reasoning loop◻ M6–M7

reason → reflect → synthesize, feeding each synthesis back as the next thought (the ouroboros), until it converges or any of four guards halts it.

start(run, prompt, config)
↓
GUARDS — checked each loop, halt at the step boundarykill · tokens≥budget · depth≥max · loops≥guard
↓ pass
reason(thought) → reflect(reasoning) → synthesize()emit a step event per transition → monitor + run_steps
↓
tokens += used · energy = decay(energy)energy strictly decreasing
↓
converged? or energy < floor?
↓ no
↺ thought ← synthesis · depth+1 · loops+1 → back to GUARDS
↓ yes
complete(run, stop_reason)

Termination guaranteed by four independent halts and monotonic energy decay — no input loops forever. Steer is checked only at a boundary, so injected guidance never corrupts mid-transition state.

A5 · Real-time fan-out with authoritative ordering◻ M3

Sends go through REST/SSE so the server owns order; the room fans out. A slow client never stalls the rest.

# server-owned order: seq assigned inside the write txn
def append_node(branch, role, content):
    with transaction:
        seq  = branch.next_seq          # monotonic, per-branch lock
        node = persist(Node(branch, parent=branch.head, seq, role, content))
        branch.head = node              # advance cached tip
    return node

def broadcast(wid, event):
    for client in rooms[wid].clients:
        try: client.queue.put_nowait(event)   # never awaits a slow socket
        except QueueFull: drop(client)         # laggard only → no head-of-line block

# client renders by node.seq, not arrival time → one total order

Complexity: O(C) non-blocking enqueues per event. Backpressure: isolated per client (NFR-7).

A4 · Budget metering◻ M7
def on_step_tokens(run, used):
    run.tokens += used
    pct = run.tokens / budget
    emit_meter(run, pct)        # green→amber→oxblood
    if pct >= 0.85 and not run.alerted:
        run.alerted = True; alert()   # one-shot
    if run.tokens >= budget:
        request_halt(run)       # cooperative

O(1)/step · one-shot alert · halt honoured at next guard.

A7 · Invite lifecycle✅ Built · M1
def accept_invite(token, user):
    inv = invites[token]              # PK lookup
    if inv is None or inv.is_expired:
        raise 404                     # expired ≡ invalid
    if member(user, inv.ws):
        return existing            # idempotent
    add Membership(user, inv.ws, inv.role)
    return workspace(inv.ws)

O(1) · unguessable token · expiring · idempotent.
